Making a Tech Support Chatbot using Google's DialogueFlow CX

In this blog, I will document the whole process of making a Custumer Support Bot, which is the main project of my internship. I will be using google's DialogueFlow (previously called API.AI) in order to achieve this.

NOTE: You can play around with the chat bot that I've created in the bottom right corner of this page :)

Introduction: Why DialogueFlow?

There were a lot of platforms to make chat bots like Microsoft Bot Framework, or IBM Watson; but I chose DialogueFlow because it was recommended by a colleage, and agter evaluating the other frameworks I've realized that DialogueFlow would require less work and be a simpler approach. Plus DialogueFlow is free for my use case.

What's the Flow bro?

We first start by setting up our flow and how we're going to make this bot work logically. This is ironically not done on DialogueFlow, but instead is done on pen and paper. This is the flow I chose to make for a fictional Technical Support Bot agent that can be implemented in a website.

After Establishing the flow, we can move on to actually implementing it on DialogueFlow. To make the implementation easier and avoid hours of debugging, I recommend starting with the moving pieces first, meaning the intents (which is what the bot thinks the user wants) and the entities (similar to Parameters)
